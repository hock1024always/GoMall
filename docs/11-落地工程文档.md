# Gomall 项目落地工程文档

## 一、项目概述

Gomall 是一个基于云原生架构的电商后端项目，采用微服务架构设计，实现了完整的电商业务流程。项目在原有基础上进行了全面重构，实现了以下核心优化：

1. **Saga分布式事务** - 解决跨服务数据一致性问题
2. **Casbin权限管理** - 实现细粒度的RBAC权限控制
3. **Redis主存储优化** - 提升高并发场景性能
4. **Nginx+OpenResty网关** - 秒杀场景请求过滤
5. **OLTP/OLAP双架构** - 支持实时交易和数据分析

## 三、核心功能实现

### 3.1 Saga分布式事务

#### 实现位置
- `common/saga/` - Saga核心模块
- `app/checkout/biz/saga/` - Checkout Saga实现

#### 执行流程
```
用户请求 → Checkout Service → Saga 协调器
  │
  ├─ 步骤1: 扣减库存 (Redis Lua脚本保证原子性)
  │    └─ 补偿: 恢复库存
  │
  ├─ 步骤2: 创建订单 (Order Service)
  │    └─ 补偿: 取消订单
  │
  ├─ 步骤3: 清空购物车 (Cart Service)
  │    └─ 补偿: 恢复购物车
  │
  ├─ 步骤4: 支付 (Payment Service)
  │    └─ 补偿: 退款
  │
  └─ 步骤5: 标记订单已支付 (Order Service)
       └─ 补偿: 取消订单状态
```

#### 使用示例
```go
// 查询Saga状态
resp, err := checkoutClient.GetSagaStatus(ctx, &checkout.GetSagaStatusReq{
    SagaId: "saga-uuid",
})
fmt.Printf("Status: %s, CurrentStep: %d\n", resp.Status, resp.CurrentStep)
```

### 3.2 Casbin权限管理

#### 实现位置
- `common/permission/` - 权限管理公共模块
- `app/auth/biz/middleware/` - 权限中间件

#### 角色权限矩阵

| 资源/操作 | admin | seller | customer |
|-----------|-------|--------|----------|
| product:create | ✓ | ✓ | ✗ |
| product:read | ✓ | ✓ | ✓ |
| product:update | ✓ | ✓ | ✗ |
| product:delete | ✓ | ✗ | ✗ |
| order:create | ✓ | ✗ | ✓ |
| order:read | ✓ | ✓ | ✓ |
| order:update | ✓ | ✗ | ✓ |
| cart:* | ✓ | read | ✓ |
| payment:create | ✓ | ✗ | ✓ |
| payment:refund | ✓ | ✗ | ✗ |

#### 使用示例
```go
// 初始化权限管理器
pm, err := permission.InitPermission(db, "conf/casbin")

// 检查权限
err := pm.CheckPermission(ctx, userID, "customer", "order", "read")

// 为用户分配角色
err := pm.AddRoleForUser(userID, "seller")
```

### 3.3 Redis主存储优化

#### 实现位置
- `config/redis/` - Redis配置文件
- `common/redis/` - Redis客户端封装

#### 架构设计
```
┌─────────────┐     ┌─────────────┐
│   Redis     │────▶│   Redis     │
│   Master    │     │   Slave     │
│  (写操作)    │     │  (读操作)    │
└──────┬──────┘     └──────┬──────┘
       │                   │
       │    ┌──────────────┤
       │    │              │
       ▼    ▼              ▼
┌─────────────────────────────────┐
│      Redis Sentinel (x3)        │
│         (故障检测&主从切换)        │
└─────────────────────────────────┘
```

#### 库存管理
```go
// 初始化库存管理器
stockMgr := stock.NewStockManager(redisClient)

// 扣减库存（原子操作）
success, err := stockMgr.DecreaseStock(ctx, productID, quantity)

// 恢复库存（补偿）
err := stockMgr.IncreaseStock(ctx, productID, quantity)

// 设置秒杀状态
err := stockMgr.SetStockStatus(ctx, productID, "finished")
```

#### 购物车管理
```go
// 初始化购物车管理器
cartMgr := cart.NewCartManager(redisClient)

// 添加商品
err := cartMgr.AddItem(ctx, userID, &cart.CartItem{
    ProductID: productID,
    Quantity:  quantity,
})

// 获取购物车
items, err := cartMgr.GetCart(ctx, userID)
```

### 3.4 Nginx+OpenResty网关

#### 实现位置
- `config/nginx/nginx.conf` - Nginx配置
- `config/nginx/lua/` - Lua脚本

#### 秒杀过滤流程
```
请求 → OpenResty
         │
         ▼
    ┌─────────────┐
    │ 频率限制检查  │ (单IP每秒最多5次)
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ 商品状态检查  │ (Redis从库)
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ 库存检查     │ (Redis从库)
    └──────┬──────┘
           │
           ▼
      后端服务
```

#### 配置说明
```nginx
# 秒杀接口
location /api/checkout/seckill {
    access_by_lua_file lua/seckill_filter.lua;
    proxy_pass http://frontend;
}

# 商品缓存
location /api/product {
    access_by_lua_file lua/product_cache.lua;
    proxy_pass http://frontend;
}
```

### 3.5 OLTP/OLAP双架构

#### 实现位置

- `common/events/` - 事件发布/订阅

#### 数据流向
```
OLTP (MySQL)                    OLAP (ClickHouse)
    │                                  │
    │  ┌─────────────────────────────┐│
    └──▶       NATS 消息队列         ◀──┘
        └──────────────┬──────────────┘
                       │
                       ▼
               ┌───────────────┐
               │  Event        │
               │  Subscriber   │
               └───────┬───────┘
                       │
                       ▼
               ┌───────────────┐
               │  ClickHouse   │
               │  写入         │
               └───────────────┘
```

#### 分析查询示例
```go
// 查询销售趋势
trends, err := chClient.QuerySalesTrend(ctx, startDate, endDate)
```

### 5.1 Redis配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
