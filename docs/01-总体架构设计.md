# 总体架构设计

## 项目概述

Gomall 是一个基于 Kitex 微服务框架的电商后端项目，采用多种架构模式和技术栈来优化性能和可靠性。

## 架构设计原则

1. **OLTP/OLAP 分离**：事务型操作与分析型操作使用不同的数据库实例
2. **热数据 Redis 主存储**：高频访问数据使用 Redis 作为主存储
3. **分布式事务 Saga**：解决跨服务数据一致性问题
4. **网关层优化**：使用 Nginx + OpenResty 进行请求过滤和负载均衡
5. **权限控制**：使用 Casbin 实现细粒度权限管理

## 数据流向

### OLTP 数据流（事务型）
```
用户操作 → 微服务 → MySQL (主存储) → Redis (缓存)
```

### OLAP 数据流（分析型）
```
MySQL → NATS → ClickHouse (异步同步)
```

### 热数据流（Redis 主存储）
```
用户操作 → 微服务 → Redis (主存储) → MySQL (异步备份)
```

## 技术栈

- **RPC 框架**: Kitex
- **Web 框架**: Hertz
- **数据库**: MySQL 8.0 (OLTP), ClickHouse (OLAP)
- **缓存/主存储**: Redis 7.0 + Redis Stack
- **消息队列**: NATS
- **网关**: Nginx + OpenResty
- **服务发现**: Consul
- **分布式事务**: Saga 模式（自研协调器）
- **权限管理**: Casbin
- **监控**: Jaeger (链路追踪), Grafana (监控面板)

## 部署架构

### 开发环境
- 所有服务部署在同一台机器
- 使用 Docker Compose 管理服务

### 生产环境（模拟）
- 网关层：Nginx + Redis 从库（同机部署）
- 应用层：微服务集群
- 数据层：MySQL 主从、Redis 主从集群、ClickHouse 集群
